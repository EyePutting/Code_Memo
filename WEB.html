웹 호스팅

웹 호스팅을 가장 쉽게 하는 방법은 Git Hub를 이용하는 것이다.
호스팅 방법은 다음과 같다.

  1. Git Hub에 로그인한다.
  2. Repository를 새로 생성한다. 생성 시 Repository name은 주소 이름이 될 것임을 염두에 두고 Add a README file을 체크한다.
  3. 이후 Add file을 클릭해서 코드 파일들을 추가한다.
  4. 코드 파일이 잘 업로드 되었는지 확인한 후 Settings > Pages에 들어간다.
  5. 이후 Source에서 branch를 선택한 후 Save를 클릭한다.
  6. 잠시 로딩 시간(최대 20분)을 가진 후 생성된 링크에 접속한다.
*이때 404가 나올 경우 이유는 다음 중 하나일 가능성이 크다.
  • 아직 변경사항이 반영되기 전(로딩 중)이다.
  • main branch 내에 index.html 파일이 존재하지 않는다.
  • URL에 무언가 잘못 기입했다. ()

웹 서버 호스팅

with Web Server for Chrome

웹 서버 호스팅을 위해서는 Web Server for Chrome을 설치해야 한다. 이후 호스팅 방법은 다음을 따른다.

  1. 앱이 설치가 완료되면 앱을 실행한다.
  2. CHOOSE FOLDER을 클릭한 뒤 호스팅하고자 하는 코드 파일이 포함된 폴더를 선택한다.
  3. CHOOSE FOLDER 위에 Web Server가 켜져 있었다면 끄고 다시 켠다.
  4. 아래 뜬 URL을 복사한 후 크롬 링크창에 붙여넣기 하고 슬래시+호스팅하고자 하는 코드 파일명을 입력한다. (ex. (URL)/코딩 메모장.html)
  5. 호스팅이 잘 되었는지 확인한다.
* 위와 같은 호스팅 방식은 Web Server for Chrome을 켜둔 상태에서만 가능하다.

with Apache

다음 방법은 Bitnami 설치가 필요하다. 이후 호스팅 방법은 다음을 따른다.

  1. 앱 설치 시, 필수적인 체크요소만 체크하고 중간에 한번 더 나오는 체크는 해제한다.
  2. 앱이 어디에 설치되는지 경로를 잘 확인하고 설치 후 경로를 찾아간다.(C드라이브 > Bitnami > wampstack > apache2 > htdocs)
  3. 링크창에 http://127.0.0.1/index.html라고 적으면 bitnami 페이지가 뜨는 걸 확인한다.
  4. 이후 경로에서 찾은 index.html 파일을 수정해보고 해당하는 파일인지 확인한다.
  5. 확인 후 index.html 파일이 포함된 폴더의 다른 파일들을 모두 없애고 자신이 작성한 코드 파일들을 복붙한다.
Web Server for Chrome 방식과 Apache 방식의 차이는, 웹 서버가 개입을 하는가에 대한 여부.

with Web Server for Chrome

마찬가지로 Web Server for Chrome 앱을 사용한다.

  1. 앱을 실행한다.
  2. Web Server을 종료시켜두고 하단에 Accessible on local network에 체크표시를 켠다.
  3. 다시 Web Server을 실행한다.
  4. 정상 작동하는지 확인하기 위해 같은 와이파이에 접속시켜둔 기기로 링크에 접속한다.

with Apache

먼저 코드 파일을 가진 컴퓨터의 IP 주소가 필요하다. 윈도우에서 확인 방법은 다음과 같다.

  1. 컴퓨터 오른쪽 하단에 있는 네트워크 표시 아이콘을 우클릭하여 '네트워크 및 인터넷 설정 열기'를 클릭한다.
  2. 이후 Wi-Fi > 네트워크 및 공유 센터 > 연결 옆의 파란 링크(ex. W-Fi) > 자세히 > IPv4 주소 확인.
